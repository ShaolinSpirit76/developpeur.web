▪ Projet 6 (août 2019) : Réalisation d'une application de gestion de patient pour un hôpital

Après l’intégration de la Base de Données, cette application permet d’inscrire de nouveaux patients, de récupérer leurs informations personnelles, de les modifier et de les supprimer. On peut aussi afficher les patients dans une liste et les chercher selon leur nom grâce à une barre de recherche. L’affichage est limité par un nombre et on peut naviguer grâce à une pagination. On peut également sélectionner un patient en particulier, lui ajouter un rendez-vous, et voir le détail des rendez-vous classés selon leur ordre chronologique. On peut les modifier et les supprimer de la base de données.

Cette application a été réalisée en respectant l’architecture de type MVC (model, view, controller) et en Programmation Orientée Objet.

Rôle : Développeur Full-Stack, travail solo.

=> Projet réalisé en 4 jours.

Moyens utilisés :
• Editeur : Visual Studio Code
• Versioning : Github
• Serveur LAMP : Linux Apache2 MySQL PHP
• Gestion de la base de données : PhpMyAdmin
• Architecture : MVC
• Programmation : PDO
• Langages : HTML5, CSS3, JavaScript 1.9 et PHP 7.2
• Framework : Bootstrap 4

▪ Projet 4 (juin 2019) : Réalisation de 2 sites en PHP à partir de base de données en XML

- Réalisation d'un site pour une entreprise du BTP à partir d'une base de données XML.

Rôle : Développeur Full-Stack, au sein d'une équipe de 3.

=> Projet réalisé en 1,5 jours.

- Réalisation d'un site rencensant les actualités sur le thème des nouvelles technologies. Pour afficher les différents articles d'actualités, apparaissant du plus récent au plus ancien selon leur date de publication, il nous a fallu « parser » les fichiers XML récupérés en ligne avec PHP, ce qui nous a permis de générer de façon dynamique les différentes pages du site. Nous avons aussi travaillé sur la réécriture d’URL qui permet une lecture plus lisible des adresses URL et de n’afficher que les rubriques présentes dans le fichier XML. Puis nous avons personnalisé notre page erreur 404 en modifiant le fichier caché .htaccess. La navbar de notre site permettait de s’inscrire et de modifier ses préférences d’affichage (couleurs), gardées en mémoire par le navigateur grâce à la mise en place de cookies.

Rôle : Développeur Full-Stack, au sein d'une équipe de 3.

=> Projet réalisé en 2 jours.

Moyens utilisés :
• Editeur : Netbeans
• Versioning : Bitbucket
• Langages : HTML5, CSS3, JavaScript 1.9 et PHP 7.2
• Bibliothèque : Materialize1
• Framework : Bootstrap 4

▪ Projet 3 (mai 2019) : Création d'un site e-commerce pour un commerce du Havre en AngularJS

Création d'un site e-commerce (avec prise en compte des frais de port) pour un commerce existant sur Le Havre (Le Démon du Jeu), avec un minimum de 3 catégories pour un total de 24 produits. Comportant une page d'accueil, une navbar, le panier dans une fenêtre modale, et un footer contenant les mentions légales, les liens réseaux-sociaux, les contacts, ainsi qu'un lien google-map. Les produits comportent une image, une description, un prix et une référence. Ces informations réapparaissent dans le panier (à l'exception de la description), dans lequel il est possible de modifier la quantité ou de supprimer le produit. Le prix final s'affiche en fonction de la zone de livraison.

Le site a été développé en single-page application.

Rôle : Chef de projet Front-End, au sein d'une équipe de 4.

=> Projet réalisé en 3 jours.

Moyens utilisés :
• Editeur : Atom
• Versioning : Github
• Langages : HTML5, CSS3 et JavaScript 1.9
• Frameworks : Bootstrap 4 et AngularJS 1.7
